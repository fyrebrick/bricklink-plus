(inner) getOrders(directionopt, statusopt, filedopt) → {Promise.<order_item_resource>}

Source:
  • Order.js, line 10
This method retrieves a list of orders you received or placed.
Examples
//Retrieves a list of received orders
getOrders();
getOrder({}); //is the same as above
//Retrieves a list of placed orders
getOrders({direction:"out"});
//Retrieves PENDING or COMPLETED received orders
getOrders({status:"pending,completed"});
//Retrieves a list of received orders not in PURGED status
getOrders({status:"-purged"});
//Retrieves a list of received and filed orders in COMPETED status
getOrders({filed:true,status:"completed"});
Parameters:
Name Type Attributes Default Description
direction string <optional>
 "in" The direction of the order to get. Acceptable values are:
  • "out": Gets placed orders.
  • "in": Gets received orders. (default)
status string <optional>
 The status of the order to include or exclude.
  • If you don't specify this value, this method retrieves orders in any status.
  • You can pass a comma-separated string to specify multiple status to include/exclude.
  • You can add a minus(-) sign to specify a status to exclude
filed boolean <optional>
 false Indicates whether the result retries filed or un-filed orders. Acceptable values are:
  • "true"
  • "false": (default)
Returns:
If successful, this method returns a list of the the summary of an order resource as "data" in the response body.
Type
Promise.<order_item_resource>

Source:

This method updates properties of a specific order. Cost fields will only be updated if the order is in unpaid status. Otherwise they will be ignored. If sales tax are to be collected by BL, every change made to the costs will recalculate the sales tax.
Example
//Updates order #1234
updateOrder(1234,{
    cost:{
        shipping: 5.0
        insurance: 2.5
    },
    remarks:"order updated"
});
